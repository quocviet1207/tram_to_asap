# TRAM → ASAP: Motion Retargeting to Humanoid in IsaacGym

A complete step-by-step guide to extract SMPL motions from [TRAM](https://github.com/yufu-wang/tram)
and retarget them to a humanoid robot (Unitree G1) using [ASAP](https://github.com/LeCAR-Lab/ASAP)
for training in IsaacGym.

---

## Overview

```
TRAM (video → SMPL)
        ↓
  hps_track_0.npy + camera.npy
        ↓
  convert_tram.py        ← YOU WRITE THIS (Phase 3)
        ↓
  IMG_7483_track0.npz    (ASAP SMPL format, coordinate-corrected)
        ↓
  fit_smpl_shape.py      (already in ASAP — one-time per robot)
        ↓
  fit_smpl_motion.py     (already in ASAP)
        ↓
  0-motions_raw_..._IMG_7483_track0.pkl   (G1 DOF motion)
        ↓
  train_agent.py         (IsaacGym RL training)
```

## Understanding the TRAM Output Keys

> ⚠️ The actual key names differ from what TRAM's documentation suggests.

```python
hps = np.load('hps_track_0.npy', allow_pickle=True).item()
# Actual keys: ['pred_cam', 'pred_pose', 'pred_shape', 'pred_rotmat', 'pred_trans', 'frame']

# pred_pose:    torch.Size([T, 144])      — 6D rotation (24 joints x 6), NOT axis-angle
# pred_shape:   torch.Size([T, 10])       — SMPL betas per frame
# pred_rotmat:  torch.Size([T, 24, 3, 3]) — rotation matrices  ← USE THIS, not pred_pose
# pred_trans:   torch.Size([T, 1, 3])     — camera-space translation  ← needs .squeeze(1)
# pred_cam:     torch.Size([T, 3])        — weak-perspective cam params
# frame:        torch.Size([T])           — frame indices

camera = np.load('camera.npy', allow_pickle=True).item()
# Actual keys: ['pred_cam_R', 'pred_cam_T', 'world_cam_R', 'world_cam_T',
#               'img_focal', 'img_center', 'spec_focal']

# world_cam_R:  (T, 3, 3) — use this (NOT 'R')
# world_cam_T:  (T, 3)    — all zeros for static camera
```

The ASAP `.npz` format requires these exact keys and types:

```
poses:           shape=(T, 72),  dtype=float32
trans:           shape=(T, 3),   dtype=float32
betas:           shape=(16,),    dtype=float64  ← 16 not 10! pad with zeros
gender:          shape=(),       dtype=<U7      (unicode string)
mocap_framerate: shape=(),       dtype=int64
```

## Coordinate System — The Critical Fix

This is the most important part to get right. There are two separate fixes needed:

### Fix 1: Translation axis remap

TRAM `pred_trans` is in **camera space** with a static camera. Inspecting `world_cam_R` reveals:

```
world_cam_R ≈ [[ 1,  0,  0],
               [ 0, -1,  0],   ← Y is flipped (camera Y=down)
               [ 0,  0, -1]]   ← Z is flipped (camera Z=depth)
```

| TRAM camera axis | Real meaning | Maps to AMASS world |
|-----------------|-------------|-------------------|
| `cam_X ≈ -0.16` | 0.16m to the left | `X` (right) — unchanged |
| `cam_Y ≈ +0.43` | person 0.43m **below** camera | `-cam_Y → Z` (up) |
| `cam_Z ≈ +6.4`  | person 6.4m **in front** (depth) | `cam_Z → Y` (forward) |

Then center XY on frame 0 so the person starts at origin (matching AMASS convention).

### Fix 2: Root rotation correction

Comparing the root euler angles (xyz degrees) between a known-good AMASS file and our raw output revealed:

```
Reference AMASS standing pose root euler:  [ 90.5°,  -9.4°,   0.2°]
Our raw TRAM output root euler:            [-177.0°,   9.2°,  -1.8°]
```

A standing SMPL body always has X euler ≈ +90° in AMASS convention. Our value was ≈ -177°.
The fix is to **pre-multiply the root rotation matrix by R_x(-90°)**:

```
R_x(-90°) = [[1,  0,  0],
             [0,  0,  1],
             [0, -1,  0]]
```

This only applies to **joint 0** (root). Body joints 1–23 are in local frame and need no change.

**Verified output after both fixes (frame 0):**
```
Root euler (fixed): [ 90.5°   9.2°  -1.8°]   ← X≈90° = upright ✓
Reference:          [ 90.5°  -9.4°   0.2°]
```

### 3.2 Create `~/Documents/ASAP/convert_tram.py`

```python
# ~/Documents/ASAP/convert_tram.py
import numpy as np
from scipy.spatial.transform import Rotation as R

def to_np(x):
    return x.numpy() if hasattr(x, 'numpy') else np.array(x)

def tram_to_asap_npz(hps_path, camera_path, output_path):
    hps    = np.load(hps_path,    allow_pickle=True).item()
    camera = np.load(camera_path, allow_pickle=True).item()

    rotmat     = to_np(hps['pred_rotmat'])            # (T, 24, 3, 3)
    betas      = to_np(hps['pred_shape'])             # (T, 10)
    pred_trans = to_np(hps['pred_trans']).squeeze(1)  # (T, 3) <- squeeze dim 1
    T          = rotmat.shape[0]
    # world_cam_T is all zeros (static camera) — no translation correction needed

    # -------------------------------------------------------
    # FIX 1: TRANSLATION axis remap — camera space → AMASS world
    #
    #   cam_X (+right)  →  AMASS X (+right)     unchanged
    #   cam_Z (+depth)  →  AMASS Y (+forward)   direct
    #   cam_Y (+down)   →  AMASS Z (+up)         negate
    #
    # Then center XY on frame 0 (AMASS convention: person starts at origin)
    # -------------------------------------------------------
    amass_trans = np.zeros((T, 3), dtype=np.float32)
    amass_trans[:, 0] =  pred_trans[:, 0]   # cam X  → AMASS X
    amass_trans[:, 1] =  pred_trans[:, 2]   # cam Z  → AMASS Y (depth → forward)
    amass_trans[:, 2] = -pred_trans[:, 1]   # -cam Y → AMASS Z (down → up)

    # Center XY so person starts at origin
    amass_trans[:, 0] -= amass_trans[0, 0]
    amass_trans[:, 1] -= amass_trans[0, 1]

    # Floor snap Z
    amass_trans[:, 2] -= amass_trans[:, 2].min()

    # -------------------------------------------------------
    # FIX 2: ROOT ROTATION (joint 0 only)
    #
    # Debug comparison showed:
    #   AMASS standing root euler xyz: [ 90.5°,  0°,  0°]
    #   Our raw TRAM root euler xyz:   [-177.0°, 0°,  0°]
    #
    # Fix: pre-multiply root rotmat by R_x(-90°)
    #
    #   R_x(-90°) = [[1,  0,  0],
    #                [0,  0,  1],
    #                [0, -1,  0]]
    #
    # Joints 1-23 are in local frame — no change needed
    # -------------------------------------------------------
    R_fix = np.array([
        [1,  0,  0],
        [0,  0,  1],
        [0, -1,  0],
    ], dtype=np.float64)  # R_x(-90 degrees)

    rotmat_fixed = rotmat.copy()
    rotmat_fixed[:, 0] = R_fix @ rotmat[:, 0]  # (3,3) @ (T,3,3) broadcasts correctly

    # All 24 joints → axis-angle (T, 72)
    pose_aa = R.from_matrix(
        rotmat_fixed.reshape(-1, 3, 3)
    ).as_rotvec().reshape(T, 72).astype(np.float32)

    # ASAP betas: (16,) float64 — pad our (10,) with zeros
    betas_16 = np.zeros(16, dtype=np.float64)
    betas_16[:10] = betas.mean(axis=0).astype(np.float64)

    # -------------------------------------------------------
    # Diagnostic: verify root euler is close to [90, 0, 0]
    # -------------------------------------------------------
    our_root_euler = R.from_rotvec(pose_aa[0, :3]).as_euler('xyz', degrees=True)
    print("=== Root euler after fix (frame 0) ===")
    print(f"  Ours (fixed): {our_root_euler.round(1)}")
    print(f"  Reference:    [ 90.5  -9.4   0.2]")
    print(f"  X should be close to 90 for upright standing")

    print("\n=== Translation (frame 0) ===")
    print(f"  amass_trans[0]: {amass_trans[0].round(4)}")
    print(f"  Reference:      [-0.0096 -0.0006  0.9640]")
    print(f"  XY should be near 0 (centered), Z = hip height (~0 after floor snap)")

    print("\n=== Translation range ===")
    for i, ax in enumerate(['X (right)', 'Y (forward)', 'Z (up/height)']):
        print(f"  {ax}: {amass_trans[:,i].min():.3f} -> {amass_trans[:,i].max():.3f}  delta={amass_trans[:,i].ptp():.3f}")

    save_dict = {
        'poses':           pose_aa,
        'trans':           amass_trans,
        'betas':           betas_16,
        'gender':          np.str_('neutral'),
        'mocap_framerate': np.int64(30),
    }

    np.savez(output_path, **save_dict)
    print(f"\nSaved {T} frames -> {output_path}.npz")

    print("\nVerifying keys/dtypes:")
    check = np.load(output_path + '.npz', allow_pickle=True)
    for k in ['trans', 'gender', 'mocap_framerate', 'betas', 'poses']:
        v = check[k]
        print(f"  {k}: shape={v.shape}, dtype={v.dtype}")


tram_to_asap_npz(
    hps_path    = '/home/viet/Documents/tram/results/IMG_7483/hps/hps_track_0.npy',
    camera_path = '/home/viet/Documents/tram/results/IMG_7483/camera.npy',
    output_path = 'humanoidverse/data/motions/raw_tairantestbed_smpl/IMG_7483_track0'
)
```

| Value | Good | Bad |
|-------|------|-----|

| Joint std mean | > 0.10 | ~0 → pose is static, no body motion |
| Active DOFs | >= 20 / 72 | < 5 → barely any motion |

| Symptom in MuJoCo | Root euler X | Fix |
|-------------------|-------------|-----|

| Robot face-down | ~0° or ~180° | Check translation Z axis remap |
| Robot face-up | ~180° | Combine X and Z rotation fix |

## Common Errors and Fixes

| Error | Cause | Fix |
|-------|-------|-----|
| `ImportError: libpython3.8.so.1.0` | IsaacGym cannot find conda libpython | `export LD_LIBRARY_PATH=/home/viet/anaconda3/envs/hvgym/lib:$LD_LIBRARY_PATH` |
| `KeyError: 'pose'` | Wrong hps.npy key name | Use `pred_rotmat`, not `pose` |
| `KeyError: 'R'` | Wrong camera.npy key name | Use `world_cam_R` and `world_cam_T` |
| Robot face-down in MuJoCo | Wrong axis remap or wrong root rotation | Compare root euler X vs reference: should be ~+90° |
| Robot upside-down in MuJoCo | Root rotation fix direction wrong | Flip sign of `R_fix`: try `R_x(+90°)` instead of `R_x(-90°)` |
| Robot underground (Z ~ -6) | Camera depth mapped to world Z | Use axis remap: `X→X`, `cam_Z→Y`, `-cam_Y→Z` |
| Motion file not found | ASAP prepends full folder path | Use full name: `0-motions_raw_tairantestbed_smpl_IMG_7483_track0.pkl` |
| `pred_trans` shape error | Shape is `(T, 1, 3)` not `(T, 3)` | Call `.squeeze(1)` after loading |
| `betas` shape mismatch | ASAP expects `(16,)` float64 | Pad: `betas_16 = np.zeros(16); betas_16[:10] = betas_mean` |
| Policy collapsed at eval | Too early in training | Train to 8000–12000 iterations |

## Key Lessons Learned

1. **Do not use `pred_pose`** — it is 6D rotation `(T, 144)`, not axis-angle. Always use `pred_rotmat` and convert via `scipy Rotation`.

2. **`pred_trans` is in camera space**. With a static camera `world_cam_T` is all zeros, so you only need an axis remap — not a full rigid transform.

3. **TRAM camera is Y-down, Z-depth**. Must remap axes before saving: `cam_X→X`, `cam_Z→Y`, `-cam_Y→Z`.

4. **Root rotation needs a separate fix**. After the axis remap the root euler X is still wrong (~-177° vs the required ~+90°). The fix is pre-multiplying the root rotation matrix by `R_x(-90°)`. This was discovered by comparing root euler angles against a known-good AMASS file.

5. **Only fix joint 0 (root)**. Body joints 1–23 are in local frame relative to the parent joint — they need no coordinate change.

6. **ASAP auto-names retargeted files** as `0-motions_raw_tairantestbed_smpl_<input_name>.pkl`. Always use this full generated name in training commands.

7. **Shape fitting** (`fit_smpl_shape.py`) only needs to run once per robot type, not once per motion clip.

8. **Always visualize in MuJoCo** after retargeting and before training — it takes seconds and catches all conversion bugs immediately.
